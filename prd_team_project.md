# 📋 팀 프로젝트 칸반 보드 앱 제품 요구사항 정의서 (PRD)

*   **프로젝트명**: Team Collaboration Kanban Hub (팀 프로젝트 진행 상황 정리 앱)
*   **작성일**: 2026년 6월 25일
*   **버전**: v1.0

---

## 1. 배경 및 목적 (Background & Objectives)
*   **배경**: 대학교 조별 과제나 소규모 IT 프로젝트를 진행할 때, 단톡방이나 구글 문서만으로는 *"지금 누가 무슨 일을 하고 있고, 어디까지 완료되었는지"* 파악하기가 매우 어렵습니다.
*   **목적**: 복잡하고 무거운 상용 협업 툴(지라 등) 대신, 비전공자도 1초 만에 이해할 수 있는 **직관적인 포스트잇(칸반 보드) 형태의 진행 상황 공유 웹 애플리케이션**을 제공합니다.

## 2. 핵심 타겟 사용자 (Target Audience)
1. **조별 과제 조장 (팀장)** : 전체 업무의 데드라인과 마일스톤 달성률(%)을 한눈에 보고 쪼아대고 싶은 사람
2. **프로젝트 팀원** : 본인이 맡은 업무 카드를 확인하고 완료 처리를 쉽게 공유하고 싶은 사람

## 3. 기능 요구사항 (Functional Requirements)

### 3.1. 메인 대시보드 (Kanban Board UI)
*   **3단계 상태 컬럼**: 업무를 `[할 일(TODO)]`, `[진행 중(IN_PROGRESS)]`, `[완료됨(DONE)]` 3가지 상태로 분리하여 표시해야 합니다.
*   **실시간 카운트 뱃지**: 각 상태별로 현재 몇 개의 태스크가 들어있는지 실시간 숫자 뱃지로 띄워야 합니다.
*   **원클릭 상태 이동**: 카드 하단에 좌우 화살표 버튼(`◀` `▶`)을 배치하여 클릭 한 번으로 다음 단계 혹은 이전 단계로 업무를 이동시킬 수 있어야 합니다.

### 3.2. 진척도 시각화 (Milestone Progress Bar)
*   **동적 게이지 바**: 전체 등록된 태스크 수 대비 `완료됨(DONE)` 상태인 태스크의 비율을 백분율(%)로 자동 계산합니다.
*   상단 헤더 아래에 시각적인 퍼센트 숫자와 함께 게이지 바로 진행률을 출력합니다.

### 3.3. 업무 카드 등록 및 속성 (Task Card)
*   **모달 폼 제공**: 우측 상단 `[+ 새 업무 추가]` 버튼 클릭 시 입력 폼이 팝업되어야 합니다.
*   **필수 속성**:
    *   **업무 제목**: 직관적인 한 줄 요약
    *   **담당자명**: 실명 혹은 별명
    *   **역할 태그**: `👑팀장`, `💻기획`, `🎨디자인`, `⚙️백엔드`, `🚀프론트` 중 택 1
    *   **마감일**: `D-3`, `오늘까지` 등 자유 텍스트 데드라인
    *   **상세 설명**: 세부 과제 내용 (선택)

## 4. 디자인 및 UX 원칙 (Design & UX Guidelines)
*   **프리미엄 다크 테마**: 눈이 피로하지 않은 어두운 배경(`slate-950`)에 에메랄드, 인디고 등 네온 그라데이션 포인트 컬러를 사용해 세련된 느낌 부여.
*   **초간단 인터페이스**: 전공 지식이 없는 대학교 1학년 신입생도 별도 설명서 없이 척척 누를 수 있는 명쾌한 직관성 우선.

## 5. 기술 규격 및 인프라 (Tech Stack & Architecture)
*   **Frontend / Backend**: Next.js 16 (App Router 기반 풀스택 프레임워크)
*   **Database**: SQLite (설치가 필요 없는 로컬 파일 기반 DB `dev.db`)
*   **ORM**: Prisma ORM v7 (공식 `better-sqlite3` 어댑터 적용)
*   **실행 환경**: 로컬 개발 서버 포트 **3001번** (`npm run dev -p 3001`)
