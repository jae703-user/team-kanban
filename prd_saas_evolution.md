# 🚀 제품 요구사항 정의서(PRD) : "팀 프로젝트 스마트 칸반 B2B SaaS 확장판"

> **프로젝트 정체성** : 대학교 실습작을 넘어, **실제 조직이 매일 접속해 일하는 한국형 실무 협업 플랫폼(SaaS)으로의 진화 설계서**

---

## 1. 비즈니스 개요 (Business Overview)

### 1.1 배경 및 문제점 (Problem)
* **단톡방 소통의 한계** : 기존 조별 과제 및 소규모 프로젝트 팀들은 카카오톡 단톡방에 업무 지시, 파일 첨부, 진척도 확인을 의존하고 있음. 이로 인해 대화가 묻히고 정보가 파편화되어 **업무 누락율 40%, 확인 대기 시간 일평균 25분 소요**라는 소통 병목 발생.
* **기존 유료 툴의 과도한 스펙** : 지라(Jira), 노션(Notion) 등은 훌륭하나 인당 월 1~2만 원의 구독료 부담과 복잡한 학습 곡선(Learning Curve)으로 인해 팀원들이 입력을 포기함.

### 1.2 솔루션 제품 비전 (Solution)
> **"단톡방 링크 한 번 클릭으로 입장하고, 포스트잇 드래그 한 번으로 대화가 끝나는 마찰 제로(0%) 협업 툴"**

## 2. 핵심 아키텍처 3대 기획 확정안 (Locked-In Specs)

시니어 프로그래머 컨설팅을 통해 팀장님이 직접 의사결정한 **3가지 핵심 차별화 아키텍처**입니다.

### ① 공간 아키텍처 : [다중 워크스페이스 방 입장 구조] (Notion/Slack 모델)
* **기획 내용** : 단일 보드 쓰레기통 현상을 방지하기 위해 `[📁 1학기 조별과제]`, `[📁 창업 동아리팀]` 등 독립된 워크스페이스 격리 방을 개설함.
* **기술 구현** : PostgreSQL `Workspace` 테이블 신설 및 `WorkspaceMember` 다대다 매칭 테이블 관계 설계.

### ② 소통 아키텍처 : [카드 내 실시간 피드 댓글 대화창] (Jira/Instagram 모델)
* **기획 내용** : 카카오톡 파일 첨부 이탈을 원천 봉쇄하기 위해 포스트잇 상세 모달 내에 **피드 댓글 및 파일 첨부 박제 시스템** 장착.
* **기술 구현** : `Comment` 및 `Attachment` 테이블 설계, Supabase Storage 연동 웹소켓 실시간 피드 갱신.

### ③ 바이럴 아키텍처 : [단톡방 오픈채팅형 초간단 초대 링크] (Viral Viral 모델)
* **기획 내용** : 팀장 수동 입력의 마찰을 없애기 위해 `https://kanban.app/invite/[특수-룸-키]` 링크 발급 기능 도입. 조원들이 카톡에서 링크 클릭 시 즉시 해당 워크스페이스 멤버로 도킹 완료.

## 3. 단계별 서비스 런칭 마일스톤 (Roadmap)

```text
[Phase 1 : MVP 가동 중] (✔ 완료)
 ├── Next.js 16 + Tailwind CSS 프리미엄 다크 보드 UI
 ├── Prisma v7 + Supabase 클라우드 PostgreSQL DB 연동
 └── Vercel 전 세계 상시 라이브 호스팅 배포

[Phase 2 : SaaS 뼈대 확장] (개발 예정 - 공수 약 5일)
 ├── 워크스페이스 선택 메인 대시보드 페이지 신설
 └── 단톡방 전용 초대 링크(Invite Token) 생성 로직 장착

[Phase 3 : 소통 엔진 탑재] (개발 예정 - 공수 약 4일)
 ├── 업무 카드 상세 모달 내 댓글 피드 API 개발
 └── 구글 드라이브 / Supabase Storage 산출물 파일 첨부 기능

[Phase 4 : 정식 베타 런칭]
 ├── 대학교 총학생회 및 IT 동아리 대상 무료 베타 바이럴 배포
 └── 비전공자 상사 및 투자자 VC 시연 보고
```

## 4. 기대 효과 및 발표 스토리텔링 포인트

1. **비용 절감** : 서버리스 DB 무료 티어 최적화 설계를 통해 **100명 접속 시에도 월 구독료 고정비 0원 달성**
2. **업무 속도** : 단톡방 파편화 해소로 **팀 일일 진척도 파악 대기 시간 70% 단축**
3. **보안성** : 미인증 외부인의 접근 차단 및 금융권 SHA-256 특수 암호화 100% 적용
